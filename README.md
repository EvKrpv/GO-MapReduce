# GO-MapReduce
Частотный анализ слов в огромном файле.

В процессе написания
Архитектура решения
Два этапа:

Mapper
  - Читает блок из файла.
  - Разбивает на слова, считает частоту в локальной map[string]int.
  - Отправляет результат по каналу chan map[string]int.
Reducer
  - Принимает частотные карты от mapper-ов.
  - Объединяет их в один map[string]int, при необходимости сбрасывает редкие слова.
  - Ведёт top-N частых слов через sort или heap.
